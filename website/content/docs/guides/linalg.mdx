## Available operations

### Decompositions

| Operation | Cost | Weight | Notes |
|-----------|------|--------|-------|
| <OpRef name="linalg.svd" /> | $m \cdot n \cdot k$ | 4.0 | Truncated SVD |
| <OpRef name="linalg.eig" /> | $10n^3$ | 4.0 | General eigendecomposition |
| <OpRef name="linalg.eigh" /> | $4n^3/3$ | 4.0 | Symmetric eigendecomposition |
| <OpRef name="linalg.cholesky" /> | $n^3/3$ | 4.0 | Cholesky (symmetric positive definite) |
| <OpRef name="linalg.qr" /> | $mn^2 - n^3/3$ | 4.0 | Householder QR (FMA=1) |
| <OpRef name="linalg.eigvals" /> | $10n^3$ | 4.0 | Eigenvalues only |
| <OpRef name="linalg.eigvalsh" /> | $4n^3/3$ | 4.0 | Symmetric eigenvalues only |
| <OpRef name="linalg.svdvals" /> | $m \cdot n \cdot \min(m,n)$ | 4.0 | Singular values only |

### Solvers

`solve_cost(n)` always returns `n^3` regardless of the `symmetric` or `nrhs`
parameters — those arguments exist for API compatibility but are currently
ignored in the cost model.

| Operation | Cost | Weight |
|-----------|------|--------|
| <OpRef name="linalg.solve" /> | $n^3$ | 4.0 |
| <OpRef name="linalg.inv" /> | $n^3$ | 4.0 |
| <OpRef name="linalg.lstsq" /> | $m \cdot n \cdot \min(m,n)$ | 4.0 |
| <OpRef name="linalg.pinv" /> | $m \cdot n \cdot \min(m,n)$ | 4.0 |

`inv` of a symmetric matrix returns a `SymmetricTensor`.

## Symmetric input savings

Pass a `SymmetricTensor` to get automatic cost reductions:

```python
import whest as we

with we.BudgetContext(flop_budget=10**8) as budget:
    A = we.as_symmetric(we.multiply(we.eye(10), 2.0), symmetric_axes=(0, 1))

    # solve_cost(n=10) = n^3 = 1000 FLOPs (symmetric/nrhs params are currently ignored)
    x = we.linalg.solve(A, we.ones(10))

    # inv returns SymmetricTensor
    A_inv = we.linalg.inv(A)
    print(isinstance(A_inv, we.SymmetricTensor))  # True
```

See [Exploit Symmetry Savings](./symmetry) for full details.

## Query cost before running

```python
cost = we.flops.svd_cost(m=256, n=256, k=10)
print(f"SVD cost: {cost:,}")  # 655,360

cost = we.flops.solve_cost(n=256)
print(f"Solve cost: {cost:,}")  # 16,777,216 (= 256^3; symmetric/nrhs params currently ignored)
```

## Common pitfalls

**Symptom:** Using `numpy.linalg.svd` instead of <OpRef name="linalg.svd" />

**Fix:** Operations called through `numpy` directly bypass FLOP counting. Always use `we.linalg.*`.
