# API Docs Coverage: Auto-Generation & Verification

**Date:** 2026-04-02
**Status:** Approved

## Problem

Operations exist in `_registry.py` (482 ops) but the docs don't cover all of them:
- `linalg`, `fft`, `polynomial`, and `window` submodules have no API reference pages
- `reference/operation-audit.md` is manually generated and stale
- No automated check ensures new ops get documented

## Solution: Pre-Build Script + CI Check

A single script `scripts/generate_api_docs.py` with two modes:

### Generate mode (default)

Reads `REGISTRY` from `mechestim._registry`, groups ops by module/category, and generates:

| Generated file | Content |
|---|---|
| `docs/api/linalg.md` | mkdocstrings directives for all linalg ops |
| `docs/api/fft.md` | mkdocstrings directives for all fft ops |
| `docs/api/polynomial.md` | mkdocstrings directives for polynomial ops |
| `docs/api/window.md` | mkdocstrings directives for window ops |
| `docs/reference/operation-audit.md` | Full audit table from registry |

Each file has a header: `<!-- Auto-generated by scripts/generate_api_docs.py. Do not edit manually. -->`

Existing `docs/api/counted-ops.md` and `docs/api/free-ops.md` are NOT regenerated — they stay as-is with their existing mkdocstrings directives.

### Verify mode (`--verify`)

1. Reads all non-blacklisted ops from the registry
2. Scans all `docs/api/*.md` files for mkdocstrings `:::` directives
3. Resolves which ops each directive covers (by importing the module and inspecting its public members)
4. Cross-references: every registry op must be covered by at least one directive
5. Exits 0 if all covered, exits 1 with a list of missing ops

### Generated page format

Each submodule page uses module-level mkdocstrings directives:

```markdown
<!-- Auto-generated by scripts/generate_api_docs.py. Do not edit manually. -->
# Linear Algebra

Operations from `mechestim.linalg`. Cost formulas vary per operation.

::: mechestim.linalg._svd
::: mechestim.linalg._decompositions
::: mechestim.linalg._solvers
::: mechestim.linalg._properties
::: mechestim.linalg._compound
```

mkdocstrings renders each function's docstring, which already includes numpy docstrings via `attach_docstring()`.

### Audit table format

The generated `operation-audit.md` replaces the current manual version. Same structure:
- Summary table with category counts
- Per-category sections listing all ops
- Generated from registry so it's always accurate

## Nav changes

New pages added to `mkdocs.yml` under "API Reference":

```yaml
- API Reference:
    - Counted Operations: api/counted-ops.md
    - Free Operations: api/free-ops.md
    - Linear Algebra: api/linalg.md
    - FFT: api/fft.md
    - Polynomial: api/polynomial.md
    - Window Functions: api/window.md
    - Budget: api/budget.md
    - FLOP Cost Query: api/flops.md
    - Errors: api/errors.md
```

## Docstring presentation

No changes to `attach_docstring()`. The existing format works:

```
[mechestim] Cost: numel(output) FLOPs | Category: counted_unary

--- numpy docstring ---
<full numpy docstring with params, returns, examples>
```

mkdocstrings renders this including all numpy parameter documentation.

## CI integration

```yaml
- name: Verify API docs coverage
  run: uv run python scripts/generate_api_docs.py --verify
```

## What stays unchanged

- `docs/api/counted-ops.md` and `docs/api/free-ops.md` — existing mkdocstrings pages
- `attach_docstring()` in `_docstrings.py` — existing docstring merge logic
- `scripts/numpy_audit.py` — still useful for finding unclassified numpy ops (different purpose)

## Developer workflow

1. Add op to `_registry.py` and implement in appropriate module
2. Run `uv run python scripts/generate_api_docs.py` to regenerate docs
3. CI runs `--verify` and fails if step 2 was skipped
